# Security Policy

At **Nawa Learn**, we take the security of our users and platform seriously. We are committed to ensuring a safe and secure environment for all students, educators, and administrators.

## Supported Versions

We currently provide security updates and support for the following versions of our platform:

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

*Note: Since Nawa Learn is an actively developed platform, we strongly recommend always deploying the latest version from the `main` branch.*

## Reporting a Vulnerability

If you discover a potential security vulnerability within Nawa Learn, please do **not** disclose it publicly. Instead, we kindly ask you to report it directly to our security team. 

Please send an email to **osissmanduajonggol@gmail.com** (or contact the core Nawasena Development Team directly) with the following details:

- A clear description of the vulnerability.
- Steps to reproduce the issue.
- Any relevant logs, screenshots, or code snippets.
- Potential impact of the vulnerability.

### What to expect

1. **Acknowledgment**: We will acknowledge receipt of your vulnerability report within 48 hours.
2. **Investigation**: Our team will investigate the issue and determine the severity and scope.
3. **Resolution**: We will work diligently to patch the vulnerability and release an update as soon as possible.
4. **Notification**: Once the issue is resolved, we will notify you and, if applicable, credit you for the discovery in our release notes.

Thank you for helping us keep Nawa Learn secure!
